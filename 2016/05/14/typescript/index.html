二、使用

安装:npm install -g typescript

编译:安装完typescript后在终端中通过tsc filename.ts指令编译生成js文件。(.ts为typescript文件后缀)

在web中调用，将编译好的js文件引用到页面中：<script src='filename.js'></script>

编辑器：
1.Webstorm：npm全局安装typescript后添加File Watcher，保存 .ts 文件自动生成 .js 和 .map 文件；
2.Sublime：Package Control:Install Package - Typescript
插件github地址：https://github.com/Microsoft/TypeScript-Sublime-Plugin

三、基本语法

数据类型：
布尔，数字，字符串，数组，多元数组，enum，any，void，其中：
enum：数值型

any:不确定类型，或只知道部分类型时使用

void：没有变量类型，通常用于函数返回结果中


变量声明
let ，const，const：声明后为不可改变常量，详见ES6规则


Interface
定义对象属性类型
参数：width:number;
可选参数：width?:number;
可变参数：…width:number;
可选参数和可变参数要放在参数列表最后；

描述函数类型:

Class

public,private,protected修饰符
public：class可调用，默认为public
private：除包含它的class内，class外均调用不到，继承也不行
protected：派生的实例内可以调用，实例外不行

模块
关于模块的引用：
import中的文件名与export或文件夹中文件的文件名一致，千万不要写错；
如果用 export= 导出，则用import = require（）引用

浏览器暂时不支持，import模块的ts文件需要进行编译，每个模块根据import的声明生成一个独立的js文件：
tsc --module commonjs Test.ts

模块与命名空间
命名空间：
用于结构化WebApp的代码，写在同一个命名空间里的代码会合并
可以存在多个文件里，通过–outFile合并，通过<script>标签内嵌至html页面
命名空间作用：stackoverflow上的解答
区别：模块与命名空间的区别
注意：不要在模块中使用命名空间

Mixin
用途：
混合多个类以实现多个功能同时调用
将每个类视为接口，通过implement而不是extend来实现
创建一个函数来遍历混合类的所有属性

三斜杠指令

用于：1.声明文件之间的依赖关系，指导编译器在编译过程中加入额外的文件；
2.文件输出（–out or –outFile），文件按照输入的顺序输出

从根文件开始，这些文件是列在命令行中的，或者是列在tsconfig.json中”file”的文件
使用”–noResolve” 忽略添加文件的命令


tsconfig.json
tsconfig.json文件所在文件夹为根文件夹，编译器根据文件中的option编译项目
file：如果file中没有文件，则默认包含所有文件；file中文件reference的文件也被包括进来
exclude：列出的文件不被编译；file不能和exclude同时使用，同时使用时file起作用；
