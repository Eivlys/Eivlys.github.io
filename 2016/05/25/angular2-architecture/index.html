Angular2 for Typescript官方文档翻译-语法基础-框架

2016-05-25

Angular2是一个帮助我们使用HTML和Javascript搭建客户端应用的框架。
框架由一些相互配合的库组成，其中一些库是核心的，一些是可选的。

我们用Angular形式的标记组成的HTML模板来写应用，通过component的类来管理这些模板，在service中添加应用的逻辑，并将顶层的root component作为Angular的启动加载器。

Angular在浏览器中接管，展示我们应用的内容，并根据我们给出的指令对用户交互作出响应。
当然Angular所做的不仅是这些，我么在分章节时会学习到细节。先来看下整体概述。

这个结构流程图中标出了Angular2应用中8个主要组件块：

  1. 模块
  2. 组件
  3. 模板
  4. 元数据
  5. 数据绑定
  6. 指令
  7. 服务
  8. 依赖注入

模块（Module）

Angular应用是模块化的。
一般来说我们的应用由许多模块组合而成。
一个只完成单一目标的内聚的代码块可以视为一个典型的模块。一个模块输出代码中的值，一般来讲只输出一个东西，比如class。

模块是可选的
强烈建议模块设计。TypeScript能很好地支持ES2015的模块语法，并且我们在章节中也将这个语法作为模块化的方法。这就是为什么我们把模块列在基础搭建部分。
Angular自身既不要求模块化的方法，也不挑剔语法。如果你不想的话可以不这样用。在你清晰掌握import和export声明后，每个章节会提供大量信息。

我们见到的第一个模块很有可能是输出一个 component 类的模块。component 是Angular的基础组成意义，我们写过很多，也将在下一个部分中聊到它。现在我们只要知道 component类一种要出从模块中导出的东西就可以了。

大多数应用都有 AppComponent 。通常来讲在名为 app.component.ts 的文件中会有它，我们将看到一段这样的声明：
export class AppComponent { }
这句 export 声明告诉Typescript该模块的 AppComponent 是公共的，应用中其他模块可以调用。

当我们需要引用 AppComponent 时，我们像这样引入：
import { AppComponent } from './app.component';
这段 import 声明告诉系统它可以从同一文件夹下名为 app.component 的模块中获取到 AppComponent 。模块名称（也叫模块id）通常就是不加后缀的文件名。

库模块
有一些模块是其他模块的库。
Angular本身就是多个库模块和一些npm包的集合。他们的名称以 @angular 前缀开始。每个Angular库都包含一个 barrel 模块，而 barrel模块实际上是包裹了几个逻辑上相关的私有模块而形成的公共模块。

@angular/core 库是最基础的Angular库模块，我们可从中调用到大多数我们需要的东西。
还有一些其他重要的Angular库例如 @angular/common，@angular/router 和 @angular/http。

我们用同样的方法从Angular库模块中引用需要的东西，例如从 @angular/core 模块中引入Angular的 Component 方法：
import { Component } from '@angular/core';
与之前引如 AppComponent 的语法相比较：
import { AppComponent } from './app.component';

发现了区别吗？在从Angular库模块中引用时，声明中只有模块名称 @angular/core ，没有路径。

当我们从自己的文件夹引用时，我们在文件名前面添加路径前缀。在这个例子中我们指定了相对文件路径，这说明作为数据源被引用的模块也在相同的文件夹内。如果数据源模块在其他文件夹内，我们也可以引用应用文件结构里上级或其他文件夹的路径。

重点内容

  • Angular应用是由多个模块组成的
  • 模块输出的内容包括：类（classes），方法（function），值（value），并由其他模块引用
  • 推荐把应用写成多个模块的集合，每个模块专注于完成一件事。
    写的第一个模块通常来讲是输出一个component。

组件（Component）

组件控制屏幕上我们称之为视图的一块区域。应用根目录带链接的shell，heroes的列表、编辑器，其视图都是由组件控制的。

定义组件的应用逻辑：在class中通过什么来控制视图。这个class通过属性和方法的接口与视图进行交互。

比如例子中的 HeroListConponent，可能有一个从服务器获取并返回一个heroes数组的 heroes 属性。当用户从heroes列表中点击一个hero时，它可能有一个 selectHero() 方法来定义 selectedHero 属性。

在用户使用应用的过程中，Angular会创建，更新，并销毁组件。开发者可选择性地根据Lifecycle Hooks对组件生命周期中的每一步进行操作。

我们或许会想知道谁在调用constructor？谁提供了服务参数？对于这些疑问，我们应该坚信当我们需要时，Angular会调用constructor并且传递一个合适的 HeroService 服务。
