Node在Mac系统中安装升级卸载及终极处理

2018-06-27

在从未搭建过node环境的电脑上安装

先安装nvm

curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.33.11 install.sh| bash

或者 

wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.33.11/Pinstall.sh| bash

重启终端（终端每次启动时检查node环境），输入

command -v nvm

输出 nvm说明安装成功

接下来安装node（npm会一起装上）

nvm install stable
nvm alias default stable

如果装了brew

brew install node

使用国内镜像比较快，或者去官网下载包根据提示安装也很方便

安装完成

node -v

看下是否安装成功，如果安装成功会显示版本号

如果报错（文件夹不存在或指令错误等），需要手动配置PATH

  • 配置PATH
    vim .bash_profile
    i 进入编辑模式
    添加语句 export PATH=’$PATH:/usr/local/bin/‘
    因为node默认安装在/usr/local/bin/
    :wq 保存并退出，立即生效
    再试一下 node -v

升级

sudo npm -g install npm@版本号

node -v查看是否升级成功

  • 未成功升级到指定版本，试试手动选择默认版本
    nvm ls 浏览列出的所有版本
    nvm use 8.11.3 指定版本号
    nvm alias default 8.11.3 设置默认版本
    重新打开终端 node -v

  • 如果npm升级失败，没有可选的最新版本，使用n来做升级（不会成功的）
    sudo npm install -g n
    sudo n stable
    如果版本号没有更新且nvm ls没有列出可选的版本，那就卸载重装吧

卸载（并没有想象的简单）

先试试
brew uninstall node
node -v
发现node命令还能执行，试试手动删除
find . -name “node”
find . -name “npm”
找到所有相关文件夹
rm -rf 路径
简单暴力

删完之后check一下是不是所有文件都清理干净了，再试试
node
找不到命令报错了，开心，重回第一步 在从未搭建过node环境的电脑上安装

Node环境搭建就是一场循环
反正我是不会去升级公司电脑的
