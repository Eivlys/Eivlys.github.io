GCP配置及初级使用

2018-06-30

通过代理服务器连接GCP

  • 本地配置
    .bash_profit 文件中添加环境变量：
    export = http_proxy = [代理服务器地址:端口]
    export = http_proxy = 127.0.0.1:1087
    export = https_proxy = 127.0.0.1:1087

  • GCP配置：
    gcloud config set proxy/type PROXY_TYPE （ 可选有 http, http_no_tunnel, socks4, socks5

    gcloud config set proxy/address PROXY_IP_ADDRESS
    gcloud config set proxy/port PROXY_PORT

如果代理有权限校验

gcloud config set proxy/username USERNAME
gcloud config set proxy/password PASSWORD

我只配置了本地文件就ok了

GCP Compute Engine 安装node环境

类似于在OS X上安装
先装nvm

curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.33.11/install.sh| bash

或者

wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.33.11/install.sh| bash

退出重新进入，输入

command -v nvm

输出 nvm 说明安装成功

然后安装node，npm会一起装上

nvm install stable
nvm alias default stable
node -v

GCP Compute Engine 安装git

参考：https://stackoverflow.com/questions/27460923/how-to-install-git-in-google-compute-engine

which git

先看看装没装，装了会显示 /usr/bin/git

如果没装，先执行

sudo apt-get update

然后安装更新

sudo apt-get install git

再试试

$ which git

会出现 /usr/bin/git
