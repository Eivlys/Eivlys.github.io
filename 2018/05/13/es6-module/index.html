ECMAScript6模块化

2018-05-13

引入模块化的目的

ECMAScript6引入模块化的目的是为了解决作用域的问题，同时也是代码更加有逻辑性，可以将大的程序拆分成小的代码块。模块内的变量不会被添加到全局，仅在作用域内有效；导入导出你需要的绑定，而不是把所有内容放在一个文件里。

导入语法

  • import后面的from指定模块文件的位置，可以是相对路径也可以是绝对路径，.js后缀可以省略
  • module.js仅在第一次import时被执行，执行后存储在内存中，之后引用其他模块时从内存调用
  • 不加{}为引用默认模块（指定默认模块见导出语法）
  • import语句先于模块内的其他语句执行，以保证依赖加载完成之后再执行逻辑

导出语法

  • 1.导入导出只能在函数外使用，不能放在函数内或逻辑判断中
  • 2.导入绑定的模块无法更改绑定的值，可以调用。如果需要更改值可以在被import的模块中更改，或者继承这个模块
  • 3.通过 as 指定绑定后的名称以避免命名重复；

模块加载

  1. 使用 script 标签直接引入js文件
  2. 内联模块
  3. 针对dom操作时，需要对页面进行监听，不能将调用的函数直接写在页面上
  4. 因为模块的引用只能在函数外使用，如要要实现有条件加载，可在if判断或回调中使用import()
