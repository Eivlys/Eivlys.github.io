ng-if 和 ng-show
当判断为否定时，ng-if将元素从dom中移去，而ng-show将元素隐藏（ 等价于CSS{display:none}）；
如果在渲染模板同时传值到页面，ng-show的模板中可以获取到值，而ng-if中的模板获取不到值；
ng-if相对于ng-show对浏览器性能消耗更大
https://stackoverflow.com/questions/19177732/what-is-the-difference-between-ng-if-and-ng-show-ng-hide

$state.transitionTo() 和 $state.go()
共同点：都用于state的跳转，都返回Promise对象
区别：$state.go()是更高级的方法。$state.go跳转失败时（如state更新但url不变），尝试使用$state.transitionTo()可以跳转成功。$state.go()从内部调用$state.transitionTo()并自动设置{ location: true, inherit: true, relative: $state.$current, notify: true }，使用时只需要更改toParam参数。

单次绑定
在变量名前加双冒号.
变量值获取完全后则停止监听，若值为空则继续监听直到获取到值为止。
目的：数据绑定后释放资源，减少监听数量，加快页面渲染速度；
官方说明：https://docs.angularjs.org/guide/expression#-event-
用法：https://toddmotto.com/angular-one-time-binding-syntax/

一些问题

fn() is not a function
检查fn()是否与变量重名，若重名fn()可能会被赋值为变量；

ng-checked 为true时仍然未选中
删掉 input 里的 ng-model！！！！！checkbox不要加ng-model！！！！！ OTZ（这个问题改了两天）

require.js:166 Uncaught Error: Load timeout for modules
require.config(）中添加 ‘waitSeconds: 0’不设置等待时间限制

ng-change is throwing error [$compile:ctreq]
在元素中使用ng-change时也同时要声明ng-model；使用了ng-click的select元素没有声明ng-model是也会出现同样的问题

在md.dialog中操作后页面不跳转
把 state.go() 写在 md.dialog的回调里

火狐浏览器中input框的$event.srcElement取不到值
改为$event.target，火狐，chrome都兼容

用$cookieStorage.put()添加cookie后，取到的cookie值带引号
如$cookieStorage.put(‘co’,’value’)，取到的 co 为“value”而不是 value
改用$cookies.put()，存取值正常
