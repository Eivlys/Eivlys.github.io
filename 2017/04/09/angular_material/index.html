不要在Web中使用Angular Material

2017-04-09


新版的Angular Material首页满是Tiffany蓝，很是粉嫩呢。

Angular Material分明是一个标准适用于移动端的UI框架，尽管介绍里写的’Hit the ground running with comprehensive, modern UI components that work across the web, mobile and desktop’，但对于Web也仅仅是work，而不是fit，举例：

1.按钮大得跟饼一样

大按钮5656，小按钮4444，正好是食指在移动设备上的触点面积。

2.输入框只有一条线

移动端的界面交互中倾向于减少用户输入操作，因此弱化输入框，强化点击按钮。

3.于是slider看起来就像饼放在拉面上（是的，有些饿了）

4.最重要的一点是：没有table样式！

第一次接触Angular Material的时候，我把文档前前后后翻了三遍只为一页table，然而失望得发现这是一个移动端的框架。Grid排版堪称鸡肋，在web端可以flex，移动端…没有那么多内容要横着排。在material风格化的页面中加表格实在画风惊奇。

UI框架有很多啊，不要妥协于Angular！
